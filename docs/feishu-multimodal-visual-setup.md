# 飞书多模态视觉识别配置指南

> 本文件说明如何利用飞书原生 AI 能力配置多模态视觉摘要字段。

## A. 如何测试飞书多模态 AI 是否真正支持附件识图

### 配置步骤

1. 打开素材库表 → 点击「AI·视觉摘要」字段 → 选择「配置自动填充」
2. 在输入源中，选择「附件」类型的字段（原图片 或 视频封面）
3. **不要选择**：巡检、地址、KOL、抠图等插件类字段
4. **不要选择**：图片链接（URL 文本），只选附件字段

### 测试方法

1. 找一条包含原图片或视频封面附件的记录
2. 在 AI·视觉摘要 字段中触发一次自动填充
3. 查看输出结果

### 判断标准

**真正识图的输出**：描述图片中具体的人物、场景、文字、颜色、构图等视觉内容。

**未真正识图的输出**：
- 只输出字段名称或文件名
- 输出与图片无关的泛泛描述
- 输出"无法识别"或空白
- 输出基于标题/作者/链接的推测性内容

如果飞书原生 AI 无法识别附件，可按下方 C 节方案切换到外部视觉流程。

## B. 可直接粘贴到多模态智能提取的视觉分析提示词

```
请分析这张图片，只看图片内容，不依赖标题或文件名。

输出格式（不超过120字）：
镜头：
主体：
光线与色调：
画面氛围：
可复用元素：

如果无媒体内容，输出：待媒体生成，暂不分析。
```

### 使用方法

1. 将上述提示词粘贴到飞书「多模态智能提取」或「AI 自动填充」的提示词输入框
2. 输入源选择：原图片、视频封面
3. 输出目标：AI·视觉摘要
4. 单次只对一条记录运行测试

### 输出示例

```
镜头：中景，正对镜头
主体：一位女性，坐在书桌前，手托下巴
光线与色调：暖色台灯，柔和自然光
画面氛围：安静、沉思、有故事感
可复用元素：桌面构图、暖光氛围、人物情绪特写
```

## C. 飞书多模态能力不足时的替代方案（本次不实施）

若飞书原生 AI 无法将附件内容传入模型，下一步可选方案：

### 方案 1：本地视觉 API 流程

1. 用 Mac mini 的 Python 脚本调用本地部署的视觉模型
2. 从飞书 API 读取附件下载 URL
3. 下载图片 → 调用视觉模型 → 获取描述
4. 将结果写回飞书 AI·视觉摘要 字段

### 方案 2：Coze 视觉流程

1. 在 Coze 工作流中增加视觉分析节点
2. 传入附件 token → Coze 调用视觉模型 → 返回描述
3. 将 Coze 输出写入飞书 AI·视觉摘要 字段

### 方案 3：飞书开放平台多模态 API

1. 调用 `/open-apis/vision/v1/analyze` 接口（若已开放）
2. 传入图片 URL 或 file_token
3. 将返回结果写入字段

以上方案均需单独评估和实施，本次仅保留 AI·视觉摘要 字段作为未来写入目标。
